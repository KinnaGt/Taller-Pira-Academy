# 🎮 Taller Pira Academy — Proyecto Unity

> **Playground de practica en unity** para el taller Unity 6 - Fundamentos y Entorno.  
> No se requiere experiencia en programación. El código ya está listo; tu trabajo es **diseñar,entender y personalizar la escena**.

---

## 📋 ¿De qué trata este proyecto?

Este repositorio es la base de trabajo del taller. Contiene un juego 2D de plataformas completo y funcional hecho en Unity donde vas a:

- **Diseñar niveles** pintando tilemaps de suelo, plataformas, escaleras y peligros.
- **Poblar la escena** con prefabs listos (jugador, enemigos, objetos interactuables, coleccionables).
- **Configurar comportamientos** desde el Inspector de Unity sin tocar una sola línea de código.
- **Personalizar** sprites, sonidos y animaciones como desafío extra.

> ⭐ **Usar los prefabs es obligatorio.** Todo lo demás es un plus.

## 🚀 Cómo arrancar desde cero

### Paso 1 — Descargar el proyecto

> No necesitás saber Git ni usar la terminal. Solo seguí estos pasos.
1. Hacé clic en el botón verde que dice **`Code`**
2. Seleccioná **`Download ZIP`**
3. Una vez descargado, **extraé** el contenido del `.zip` en una carpeta de tu disco  
   *(por ejemplo: `Documentos/MisProyectos/Taller-Pira-Academy`)*

> ⚠️ **Importante:** No intentes abrir el proyecto desde adentro del archivo comprimido. Siempre extraé primero.

---

### Paso 2 — Agregar el proyecto a Unity Hub

1. Abrí **Unity Hub**
2. Hacé clic en el botón **`Add`**  
   *(o en la flecha junto a "Open" → "Add project from disk")*
3. Navegá hasta la carpeta que acabás de descomprimir  
   Tenés que seleccionar la **carpeta raíz**: la que contiene las subcarpetas `Assets`, `Packages` y `ProjectSettings`
4. Hacé clic en **`Add Project`**

---

### Paso 3 — Instalar la versión de Unity correcta

Unity Hub detecta automáticamente qué versión del motor necesita el proyecto.

1. Si ves un aviso **"Editor version not installed"** en la lista de proyectos, no te preocupes
2. Hacé clic en el ícono de advertencia o la flecha junto a la versión de Unity
3. En la ventana que aparece, hacé clic en **`Install [Versión]`**
4. Unity Hub descargará e instalará la versión exacta *(ej. Unity 6.0.x)*  
   Esto garantiza que no tengas problemas de compatibilidad

---

### Paso 4 — Abrir el proyecto por primera vez

1. Una vez instalada la versión del Editor, hacé clic en el **Curso-Pira-Taller** en Unity Hub para abrirlo
2. **Tené paciencia:** la primera vez que se abre, Unity tiene que importar todos los archivos y genera una carpeta `Library`. Este proceso puede tardar **varios minutos**
3. Si al abrir ves la pantalla vacía, busca en la ventana **Project** la carpeta
   `Taller` y haz doble clic en la escena **Playground**.

## ❓ Preguntas frecuentes

**¿Puedo abrir el proyecto con otra versión de Unity?**  
No recomendado. Usar una versión diferente a la especificada puede generar errores difíciles de resolver. Instalá la versión exacta desde Unity Hub.

**El proyecto tardó mucho en abrir, ¿está roto?**  
No. La primera apertura siempre es lenta porque Unity procesa todos los assets. Las veces siguientes abre mucho más rápido.

**Hice algo mal y el proyecto no funciona, ¿qué hago?**  
Volvé a descargar el ZIP del repositorio y repetí los pasos. Como no modificás código, siempre podés partir de cero sin perder tu trabajo en la escena *(mientras hayas guardado la escena antes)*.

**¿Puedo trabajar en equipo en el mismo proyecto?**  
Para este taller, se recomienda que cada estudiante trabaje en su propia copia descargada del ZIP.
